## 关键教训

### 核心瓶颈：非Microsoft邮箱 + 域名限制
- **新发现**：大量V2Board/SSPANEL机场的注册表单使用域名下拉框，仅支持@qq.com/@163.com/@gmail.com，**不含@outlook.com**
- 已扫描15个机场，0个支持@outlook.com! 逗猫/一元机场也不行
- 奈云v2ny是少有的完整邮箱输入+明确列出@outlook.com的机场
- TaiShan Net使用域名下拉框但有@outlook.com选项
- FSCloud/cd520有@outlook.com但服务器500错误/拒收
- GLaDOS是唯一确认给4天真免费试用的，但拒所有Microsoft邮箱
- **结论：必须搞非Microsoft邮箱才能打开30+机场的大门**

### 🆕 新发现的注册模式
6. **域名下拉框SSPANEL** (TaiShan Net/cd520/极光加速)：邮箱=前缀+域名下拉框，需要用JS `dispatchEvent('change')+dispatchEvent('input')` 触发Vue响应。直接设select.value不够。已写 `scripts/scan_outlook_support.py` 批量扫描。

### 已验证的注册系统分类
1. **SSPANEL** (稳连云/大象/三毛/魔戒)：无验证码/需验证码，VUE_NAIVE_ACCESS_TOKEN存localStorage，订阅链接= {域名}/api/v1/client/subscribe?token={uuid}
2. **Mala-Pro/Vue SPA** (宝可梦)：需验证码，API域名和面板域名分离，subscribe_url存localStorage
3. **hidexx** (aiguobit)：SSO认证，需验证码，无订阅链接只有购买后给节点
4. **扬帆云V2Board** (1元/2元)：强反爬，register→login死循环
5. **网际快车SSPANEL** (三毛)：中转网关c.xn--66tw07h.com→内部面板

### 技术栈
- **浏览器**: playwright-cli v0.1.14 + Edge Chromium (能穿透GFW访问被墙站点！Python requests不行)
- **验证码**: Microsoft Graph API (14个token账户可用)
- **playwright-cli eval限制突破**: 逗号操作符串联表达式 `(a=1, b=2, a+b)`；不能用var/function/for
- **订阅链接通用提取公式**: 登录→eval取VUE_NAIVE_ACCESS_TOKEN→fetch /api/v1/user/info→取uuid→拼URL
- **测试脚本**: test/文件夹，主项目代码严格隔离
